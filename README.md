# ВС РФ · в/ч №12132 — сайт воинской части

Статический сайт для 1-й ОБрСпН (в/ч №12132). Хостинг — GitHub Pages. Опциональная авторизация и БД через Supabase.

## Структура (плоская, без подпапок)

```
/
├── index.html          — главная (hero, о части, статистика, уставы, новости)
├── ustav.html          — уставы (ваш готовый файл, положите сюда)
├── gallery.html        — галерея с фильтрами и lightbox
├── news.html           — лента новостей + админ-панель
├── autopark.html       — карточки военной техники
├── map.html            — заглушка карты (замените своей)
├── style.css           — все стили, темы, компоненты, адаптив
├── main.js             — фон, темы, бургер, анимации, переходы
├── auth.js             — Supabase-авторизация (опционально)
├── news.js             — загрузка новостей (Supabase → демо-fallback)
├── config.js           — ключи Supabase (пусто по умолчанию)
├── SUPABASE.md         — инструкция подключения БД
└── README.md
```

**Все файлы в корне репо.** Никаких `css/`, `js/`, `data/`. Просто загрузите содержимое папки в корень репозитория.

## Деплой на GitHub Pages

1. Создайте репозиторий и загрузите все файлы в корень (drag-and-drop через web-интерфейс работает).
2. Settings → Pages → Source: `main` (или `master`) branch, папка `/ (root)`.
3. Сохраните. Через 1–2 минуты сайт доступен: `https://<username>.github.io/<repo>/`.

Готово. Никаких дополнительных настроек не нужно.

### Если сайт не открывается или стили не грузятся

- Убедитесь, что все файлы **действительно в корне** репо (не внутри папки типа `site/`).
- Подождите 2–3 минуты после первого push — Pages кэшируется.
- Проверьте регистр имён файлов: `style.css` и `Style.css` — разные файлы на GitHub Pages (Linux).
- В DevTools (F12) → Network — посмотрите, какие пути пытается загрузить браузер.

## Ustav.html

У вас уже есть готовый `ustav.html` — просто положите его в корень. Ссылки из шапки уже настроены на него.

## Подключение Supabase (опционально)

См. [SUPABASE.md](SUPABASE.md) — пошагово: создание проекта, ключи, SQL для таблиц, RLS-политики, создание админа.

Кратко: заполните `config.js` и админка сразу заработает.

## Как добавить новую страницу

1. Скопируйте `autopark.html` как шаблон.
2. Поменяйте `<title>`, содержимое `<main>` и атрибут `data-page="..."` активной ссылки в навигации.
3. Добавьте пункт `<a href="...">` в `<nav>` во всех остальных HTML-файлах.

## Темизация

Все цвета — CSS-переменные в начале `style.css`. Правьте `--gold`, `--green1..3`, `--bg`, `--paper` — всё остальное подхватится.
